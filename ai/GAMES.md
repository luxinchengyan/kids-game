## 📍 十、游戏模式设计 (Game Modes)

### 10.1 单人模式 (Single Player Mode)

#### 模式概述
- **目标**：个人学习与技能提升
- **适用场景**：日常学习、技能训练、知识巩固
- **当前状态**：✅ 已实现（打地鼠等游戏）

#### 核心功能
| 功能 | 说明 | 实现状态 |
|-----|------|---------|
| **难度选择** | 入门/进阶/挑战三级难度 | ✅ 已实现 |
| **进度追踪** | 记录最佳成绩、总星星数 | ✅ 已实现 |
| **错题本** | 自动记录错误题目 | 📋 计划中 |
| **成就系统** | 解锁徽章和成就 | ✅ 基础实现 |
| **连续签到** | 每日学习奖励 | ✅ 已实现 |

#### 难度系统设计
```typescript
interface DifficultyConfig {
  moleCount: number;        // 同时出现的目标数
  showTime: number;         // 目标显示时间(ms)
  spawnInterval: number;    // 生成间隔(ms)
  roundCount: number;       // 总回合数
  gridSize?: number;        // 网格大小 (3x3, 4x4)
  dataPool?: string;        // 数据池类型

const difficultySettings = {
  easy: {   // 3-4岁
    moleCount: 3,
    showTime: 3000,
    spawnInterval: 1500,
    roundCount: 8,
    gridSize: 3,
  },
  medium: { // 4-5岁
    moleCount: 4,
    showTime: 2000,
    spawnInterval: 1200,
    roundCount: 10,
    gridSize: 3,
  },
  hard: {   // 5-6岁
    moleCount: 5,
    showTime: 1200,
    spawnInterval: 900,
    roundCount: 12,
    gridSize: 4,
  },

```

#### 自适应难度（AI增强）
- **动态调整**：根据正确率自动升降难度
  - 连续3次正确 → 提升难度
  - 连续3次错误 → 降低难度
  - 正确率60-80% → 保持当前难度
- **能力评估**：首次游戏进行5题能力测试
- **个性化推荐**：基于历史数据推荐合适难度

#### 模式概述
- **目标**：与他人竞技，增加趣味性
- **适用场景**：亲子互动、朋友对战、班级竞赛
- **当前状态**：📋 计划中

| 对战类型 | 参与人数 | 玩法说明 | 技术实现 |

| **亲子对战** | 2人 | 家长vs孩子（难度补偿） | WebSocket/HTTP |

#### 对战机制设计

**1. 实时对战（同步模式）**
```
玩家A                    服务器                    玩家B
  |                         |                         |
  |------ 开始游戏 -------->|                         |
  |                         |------ 开始游戏 -------->|
  |                         |                         |
  |------ 答题正确 -------->|                         |
  |                         |------ 同步分数 -------->|
  |                         |                         |
  |                         |<----- 答题正确 ----------|
  |<----- 同步分数 ---------|                         |
  |                         |                         |
  |------ 游戏结束 -------->|                         |
  |                         |------ 游戏结束 -------->|
  |<----- 结果判定 ---------|                         |
  |                         |<----- 结果判定 ----------|
```

**2. 异步对战（挑战模式）**
```
玩家A 创建挑战 → 上传成绩 → 玩家B 收到通知 → 玩家B 挑战 → 对比成绩
```

#### 难度补偿系统（亲子对战）
为保证公平性，引入难度补偿机制：

| 角色 | 难度系数 | 时间限制 | 干扰项数量 |
|-----|---------|---------|-----------|
| **孩子(3-4岁)** | 0.6x | 5秒/题 | 2个 |
| **孩子(5-6岁)** | 0.8x | 4秒/题 | 3个 |
| **家长** | 1.5x | 2秒/题 | 5个 |

#### 对战奖励
- **胜利奖励**：基础星星 × 1.5
- **连胜奖励**：额外+2星星/连胜
- **参与奖励**：即使失败也有基础星星
- **特殊成就**："首次胜利"、"连胜3场"等

## 📍 六、游戏数据追踪与评估 (Game Data Tracking)

### 6.1 核心数据指标

#### 基础指标
| 指标名称 | 说明 | 数据类型 | 记录时机 |
|---------|------|---------|----------||
| **completionTime** | 完成时长(毫秒) | number | 游戏结束 |
| **accuracy** | 正确率(0-1) | number | 每题/游戏结束 |
| **score** | 得分 | number | 实时/结束 |
| **attempts** | 尝试次数 | number | 每次操作 |
| **stars** | 获得星星数(1-3) | number | 游戏结束 |

#### 舒尔特方格专项指标
| 指标名称 | 说明 | 计算方式 | 评估标准 |
|---------|------|---------|----------||
| **totalTime** | 总完成时间 | 从第1个到最后一个数字 | 3x3: 优<15s, 良<25s, 及格<40s |
| **avgClickInterval** | 平均点击间隔 | totalTime / (n-1) | 反映搜索速度稳定性 |
| **firstClickTime** | 首次点击时间 | 开始到点击"1"的时间 | 反映初始注意力集中速度 |
| **errorCount** | 错误点击次数 | 点错数字的次数 | 反映注意力集中程度 |
| **visualSpan** | 视幅范围估计 | 基于点击位置分析 | 反映视觉广度 |
| **searchPattern** | 搜索模式 | 点击轨迹分析 | 系统性/随机性 |

#### 记忆力游戏专项指标
| 指标名称 | 说明 | 计算方式 | 评估标准 |
|---------|------|---------|----------||
| **memorySpan** | 记忆广度 | 能正确回忆的最大序列长度 | 3-4岁: 3-4个<br>5-6岁: 5-6个 |
| **retentionTime** | 保持时间 | 显示结束到开始回忆的间隔 | 3-10秒渐变 |
| **recallAccuracy** | 回忆准确率 | 正确回忆数/总数量 | ≥80%为优秀 |
| **sequenceError** | 序列错误 | 顺序错误的数量 | 0为完美 |

#### 反应力游戏专项指标
| 指标名称 | 说明 | 计算方式 | 评估标准 |
|---------|------|---------|----------||
| **reactionTime** | 反应时间(ms) | 刺激出现到响应的时间 | 3-4岁: 500-800ms<br>5-6岁: 400-600ms |
| **avgReaction** | 平均反应时 | 多次试验平均值 | 稳定性指标 |
| **minReaction** | 最快反应时 | 所有试验中最快 | 注意力最佳状态 |
| **falseStart** | 抢跑次数 | 刺激出现前响应 | 反映冲动控制 |

### 舒尔特方格游戏实现要点

```typescript
interface SchulteGridProps {
  gridSize: 3 | 4 | 5;          // 网格大小
  difficulty: 'easy' | 'medium' | 'hard';
  onComplete: (result: {
    completionTime: number;      // 完成时长(ms)
    errorCount: number;          // 错误次数
    accuracy: number;            // 准确率
    clickSequence: ClickRecord[]; // 点击序列
  }) => void;
}

interface ClickRecord {
  number: number;               // 点击的数字
  position: { x: number; y: number };  // 网格位置
  timestamp: number;            // 点击时间戳
  isCorrect: boolean;           // 是否正确
  timeFromStart: number;        // 从开始到现在的时长
}
```

**核心逻辑**：
1. 生成1-N的随机排列
2. 渲染网格，记录开始时间
3. 监听点击，验证顺序
4. 记录每次点击的位置、时间、正确性
5. 完成后计算总时长、错误次数、平均间隔
6. 根据年龄组和时长给出评分
