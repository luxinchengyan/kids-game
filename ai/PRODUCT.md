## 一、现状诊断

### 1.1 产品增长现状（基于代码库分析）

| 维度 | 现状 | 差距 |
|------|------|------|
| **留存** | streakDays 字段存在，但无推送/提醒触达机制 | 用户沉默即流失，无召回手段 |
| **传播** | `LearnProgressShare.tsx` 组件存在，但功能骨架未完整 | 缺乏病毒传播闭环，无分享激励 |
| **活跃** | 每日签到 `checkIn()` 存在，奖励单薄（仅 stars） | 缺乏每日任务、限时活动、节日内容 |
| **运营** | analytics.ts 仅本地调试，无后端事件收集 | 无数据，无法做精细化运营 |
| **变现** | 无付费墙、无会员体系 | 缺乏商业闭环 |
| **家长端** | 家长摘要组件存在，但数据浅薄 | 家长价值感弱，付费意愿难培养 |

### 1.2 核心增长瓶颈

```
获客 → [落地页转化] → 注册 → [第一次体验] → 留存 → [分享裂变] → 再获客
                                     ↑
                              当前最大漏斗缺口：
                              ① 无分享奖励机制
                              ② 无回流提醒
                              ③ 家长看不到价值 → 付费意愿低
```

## 三、留存体系（Retention）

### 3.1 三层留存模型

| 层级 | 时间窗 | 核心机制 | 目标指标 |
|------|-------|---------|---------|
| **短期留存** | Day 1-3 | 新手引导 + 即时成就 | Day1 留存 > 60% |
| **中期留存** | Day 7-14 | 连续签到奖励 + 每周任务 | Day7 留存 > 30% |
| **长期留存** | Day 30+ | 成长图谱 + 学期目标 + 家长周报 | Day30 留存 > 15% |

### 3.2 连续学习强化系统（Streak 2.0）

**现状**：`streakDays` 存在但奖励平淡。

**升级方案**：
- **连续3天**：解锁新宠物蛋（孵化倒计时，制造期待）
- **连续7天**：宠物进化 + 家长专属分享卡「孩子已连续学习7天！」
- **连续30天**：解锁新学习岛区域（稀缺感）
- **连续断签保护**：每月1次「免死金牌」，防止因1天缺失导致放弃
- **视觉化火焰动效**：连续天数越多，火焰越大越炫，增强仪式感

**实现**：扩展 `useRewardStore.checkIn()` + 新增 `StreakMilestone` 组件

### 3.3 每日任务体系（Daily Quests）

每日展示 3 个可完成的小任务，完成后叠加奖励：

```typescript
// 示例任务结构
interface DailyQuest {
  id: string;
  title: string;       // "今天学会3个拼音"
  icon: string;
  target: number;      // 目标值
  current: number;     // 当前进度
  reward: { stars: number; xp: number; bonusItem?: string };
  expiresAt: string;   // 当天23:59
}
```

任务类型：
- **学习任务**：完成 N 个拼音/数学/英语练习
- **正确率任务**：连续答对 5 题
- **时长任务**：今日学习超过 8 分钟
- **探索任务**：尝试一个新游戏

**ROI**: 高。每日任务是提升 D1/D7 留存最成熟的机制，实现成本低。

### 3.4 个性化召回推送

当用户 **超过 24 小时未登录**，触发家长端提醒：

- 微信服务号模板消息（长期方案）
- 短期方案：PWA 推送通知（`Notification API`）
- 内容个性化：「昨天小明学了拼音 ba，今天可以学 pa 啦！」

**实现**：在 `analytics.ts` 增加 `lastActiveAt` 追踪，配合后端定时任务。

### 3.5 学习成长图谱（Learning Map）

将已有的 `homeLearningJourney.ts` 数据可视化为：
- 星球探索地图（孩子视角）：解锁的星球越来越多
- 技能雷达图（家长视角）：拼音/数学/英语/语文 四维能力图

## 四、病毒传播体系（Virality）

### 4.1 传播公式

```
K-Factor = 邀请率 × 转化率
目标 K > 0.3（即每10个用户带来3个新用户）
```

### 4.2 核心传播机制

#### 4.2.1 学习成果分享卡（最高 ROI）

基于已有的 `LearnProgressShare.tsx` 深度扩展：

**分享卡类型**：

| 触发时机 | 卡片内容 | 分享渠道 |
|---------|---------|---------|
| 完成今日学习 | 「今天学会了声母B P M F！」+ 孩子头像 + 进度条 | 微信朋友圈/微信群 |
| 连续打卡 N 天 | 「已连续学习 7 天，超越 92% 小朋友！」 | 朋友圈 |
| 解锁新成就 | 「我获得了「拼音小达人」勋章！」 | 微信好友 |
| 学习周报 | 「本周学了 XX 个知识点」图文卡 | 朋友圈 |
| 期末复盘 | 「本学期进步最大的 3 项能力」 | 家长群 |

**技术实现**：
```typescript
// 使用 html2canvas 将分享卡 DOM 渲染为图片
import html2canvas from 'html2canvas';

async function generateShareImage(cardRef: HTMLElement): Promise<string> {
  const canvas = await html2canvas(cardRef, {
    scale: 2,         // 2x 清晰度
    useCORS: true,
    backgroundColor: null,
  });
  return canvas.toDataURL('image/png');
}
```

#### 4.2.2 邀请好友体系（Referral）

- **邀请人奖励**：被邀请人完成第一次学习后，邀请人获得「超级星星 × 50」
- **被邀请人奖励**：注册后直接解锁高级宠物，跳过普通宠物孵化阶段
- **邀请码机制**：每个家长账户有唯一 6 位邀请码（`ABCD12` 格式），一键复制分享
- **双向可见**：家长端可查看「我邀请的好友」列表，增强社交感

#### 4.2.3 亲子共玩挑战（Co-play Challenges）

- **双人模式**：孩子 + 家长一起抢答，家长故意答慢，让孩子赢
- **挑战卡**：生成一张「今日挑战：和爸爸比一比谁认识的汉字多！」，可发到家庭群
- 本质：**把游戏变成家庭话题**，家长主动传播

### 4.3 内容传播（Content Marketing）

- **孩子作品集**：拼音跟读录音、绘画涂色作品，家长可分享「我家宝贝的作品」
- **排行榜截图**：班级/年龄段排行（匿名），「本地区 TOP 10% 小朋友」截图
- **里程碑证书**：PDF 证书（「XXX 小朋友完成了拼音全课程」），可打印，送幼儿园展示

## 五、活跃度体系（Engagement）

### 5.1 游戏化深度设计

#### 5.1.1 宠物养成系统（Pet System）

现有代码中 `unlockedPets: []` 和 `pets: []` 已预留字段，需落地：

```typescript
interface Pet {
  id: string;
  name: string;           // "小星星"
  species: string;        // "star_cat"
  level: number;          // 1-10
  happiness: number;      // 0-100，每日学习+10
  hunger: number;         // 0-100，每日-10，学习后回满
  evolutionStage: number; // 1-3（蛋→幼体→成体）
  specialSkill: string;   // 解锁后展示特殊技能动画
}
```

**养成循环**：
- 每日学习 → 宠物快乐值 +10，饥饿值恢复
- 宠物「饿了」→ 提醒孩子来学习（最直接的召回机制！）
- 宠物升级 → 解锁新动画 / 新技能 / 新对话
- **宠物情绪可见化**：首页展示宠物当前状态，孩子会主动关心

#### 5.1.2 限时活动（Seasonal Events）

| 节日/时间点 | 活动名称 | 内容 | 奖励 |
|-----------|---------|------|------|
| 每周一 | 「新周挑战」 | 本周学习目标挑战 | 双倍星星 |
| 儿童节（6.1） | 「六一嘉年华」 | 限时游戏 + 专属皮肤 | 节日勋章 |
| 暑假期间 | 「暑期学习营」 | 每日打卡挑战，30天完成解锁 | 暑期专属角色 |
| 开学季（9.1） | 「开学冲刺」 | 拼音/数学复习挑战 | 「小学准备好了」证书 |
| 春节 | 「新年大冒险」 | 主题关卡 + 红包奖励（虚拟金币） | 春节限定宠物 |
| 寒假 | 「寒假学习营」 | 类似暑期，30天打卡 | 限定头像框 |

**实现**：新增 `useEventStore.ts` + 活动配置文件 `src/data/seasonalEvents.ts`

#### 5.1.3 成就图鉴系统（Achievement Gallery）

将现有 `collectedBadges` 扩展为可视化图鉴：

- **学科成就**：「拼音小达人」「数学小能手」「英语小明星」
- **行为成就**：「第一次学习」「连续7天」「答对100题」
- **隐藏成就**：需要特定条件触发，制造惊喜感
- **全屏庆祝动画**：解锁成就时全屏烟花 + 音效，持续3秒，强仪式感

### 5.2 内容活跃策略

#### 5.2.1 每周新内容更新

- 每周一 **新增 5-10 个知识点**（新拼音/新单词/新故事）
- 「新内容」角标在首页显示，驱动好奇心
- 家长端推送「本周新增 XXX 内容」

#### 5.2.2 故事内容扩展（StoryBook Live）

- 每周更新 1-2 个新故事，带语音朗读
- **互动故事**：故事中有 2-3 个选择点（孩子可以选择剧情走向）
- **亲子故事挑战**：家长录音朗读故事，孩子听家长声音

#### 5.2.3 难度自适应可见化

将自适应难度变化展示给孩子：「你变强了！难度+1」，带升级动效，增强成就感。

## 六、运营体系（Operations）

### 6.1 数据埋点体系（Analytics 2.0）

现有 `analytics.ts` 只在本地调试，需要接入真实数据收集：

**核心指标（北极星指标体系）**：

| 指标层级 | 指标名称 | 计算方式 | 目标值 |
|---------|---------|---------|------|
| **北极星指标** | 有效学习完成次数/周 | 完成≥3题的session数 | > 5次/用户/周 |
| **获客** | 新注册用户数 | 注册事件 | - |
| **激活** | 首日完成率 | 完成第一次完整游戏 / 注册数 | > 60% |
| **留存** | D1/D7/D30 | 次日/7日/30日回访率 | 60/30/15% |
| **变现** | ARPU | 付费收入 / MAU | - |
| **传播** | K-factor | 邀请成功数 / 邀请发出数 × 转化率 | > 0.3 |

**关键事件埋点清单**：

### 6.2 家长价值体系（Parent Value Loop）

家长是**付费决策者**，需要专门的价值感知设计：

#### 6.2.1 家长智能周报（Weekly Digest）

每周一早上 8:00，家长收到一张「孩子上周学习报告」：

```

🎯 下周推荐：继续练习翘舌音

👀 查看完整报告 →
```

#### 6.2.2 家长学习建议引擎

- 根据孩子薄弱点，每周生成 3 条「今晚可以和孩子一起做」的建议
- 例：「今天可以跟孩子一起找找家里带'b'音的东西，比如杯子、本子」
- **线下延伸**：将游戏学习与真实生活连接，增强家长认可度

#### 6.2.3 多子女档案管理

- 现有代码 `children: Child[]` 已支持，需 UI 落地
- 一个账户管理多个孩子，各自独立进度
- 切换孩子时有明显视觉区分

### 6.3 会员付费体系（Monetization）
